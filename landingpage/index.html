Features
Everything you need.
Nothing you don't.

Built for teams that want structure without ceremony. Every feature earns its place.

Custom Ticket Types

Define Bug, Feature, Task, or anything else. Each type gets its own fields, configured by you.

Assignees & Due Dates

Every ticket requires an owner and a deadline. No ambiguity about who does what, by when.

Custom Fields

Text, numbers, dates, user references, ticket links. Attach structured data to any ticket type.

Workflow Engine

Define allowed status transitions with a visual matrix. Enforce process without code.

Project Roles

Manager, Member, Reporter. Scoped per project. Admins see everything; everyone else sees what they should.

Per-Project Config

Activate specific statuses and ticket types per project. Same system, different workflows.

Setup
Running in four steps.

From zero to tracking tickets in under five minutes.

Build

Clone the repo and run cargo build --release. One binary, no Docker required.

Configure

Copy .env.example to .env. Set your admin credentials and bind address.

Launch

Run the binary. The database, schema, and admin user are created automatically on first start.

Organize

Create statuses, define ticket types, set up projects, invite your team. Start tracking work.

Architecture
Simple stack. No surprises.

Every layer chosen for reliability and zero operational overhead.

Server Actix-web (Rust)
Templates Askama (compile-time)
Database SQLite + r2d2 pool
Auth Argon2 + session cookies
Assets Embedded in binary

One binary. One file. Done.

The entire application compiles to a single executable. CSS is embedded at build time. The database is a single SQLite file with WAL mode enabled. Migrations run automatically.

There is no build pipeline, no runtime, no container orchestration. Deploy it anywhere you can run a binary.

  • No Node.js, no npm, no webpack
  • No Postgres, no Redis, no message queue
  • No Docker required (but works in one if you want)
  • Backup = copy one .db file
